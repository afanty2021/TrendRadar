## 简介
本指南面向希望在本地或容器环境中部署 TrendRadar 的用户，覆盖以下目标：
- 通过 setup-mac.sh 与 setup-windows.bat 完成环境初始化与依赖安装
- 基于 Dockerfile 构建镜像，使用 docker-compose.yml 启动服务编排，配置网络与持久化卷
- 结合 Deployment-Guide.md 的步骤，解析 docker-compose.yml 的配置项与运行模式
- 提供健康检查、日志查看与版本升级的最佳实践
- 解决常见部署问题（如端口冲突、权限不足、配置缺失）

## 项目结构
项目采用“脚本 + Docker 编排 + MCP 服务”的多部署形态：
- 本地一键脚本：setup-mac.sh、setup-windows.bat
- Docker 镜像与编排：Dockerfile、Dockerfile.mcp、docker-compose.yml、docker-compose-build.yml
- 容器入口与管理：entrypoint.sh、manage.py
- 运行脚本：start-http.sh

## 核心组件
- 本地一键部署脚本
  - macOS：setup-mac.sh，负责 UV 安装、依赖同步、配置检查与 MCP 客户端配置指引
  - Windows：setup-windows.bat，负责 Python/UV 检测、依赖同步、配置检查与 MCP 客户端配置指引
- Docker 镜像与编排
  - Dockerfile：基础镜像、依赖安装、入口脚本、环境变量与默认入口
  - Dockerfile.mcp：MCP 服务器专用镜像，暴露 3333 端口并以 HTTP 模式启动
  - docker-compose.yml：官方示例编排，包含两个服务：trend-radar（爬虫+Web）、trend-radar-mcp（MCP 服务）
  - docker-compose-build.yml：本地构建版本，从根目录 Dockerfile 构建镜像
- 容器入口与管理
  - entrypoint.sh：校验配置、根据 RUN_MODE 选择一次性执行或定时任务，并可选启动 Web 服务器
  - manage.py：容器内管理工具，提供 run/status/config/files/logs/restart/start_webserver/stop_webserver/webserver_status/help 等命令
- 运行脚本
  - start-http.sh：本地 HTTP 模式启动 MCP 服务器（便于远程访问）
- 配置与依赖
  - config/config.yaml：应用配置（爬虫、推送、平台等）
  - requirements.txt、pyproject.toml：依赖清单

## 详细组件分析

### 本地部署：setup-mac.sh 与 setup-windows.bat
- 功能要点
  - 自动检测并安装 UV（macOS：curl 安装；Windows：PowerShell/pip 安装）
  - 同步项目依赖（uv sync）
  - 校验配置文件是否存在，给出下一步 MCP 客户端配置指引
- 使用建议
  - macOS：首次运行若提示 UV 未安装，按提示重新运行脚本
  - Windows：安装完成后需重新打开终端再运行脚本，确保 PATH 生效
  - 若依赖安装失败，检查网络与 Python 版本（>=3.10）

### Docker 镜像与编排：Dockerfile、Dockerfile.mcp、docker-compose.yml、docker-compose-build.yml
- Dockerfile
  - 基于 python:3.10-slim
  - 安装依赖、复制入口脚本与 manage.py、创建 config/output 目录、设置环境变量
  - 默认 ENTRYPOINT 为 /entrypoint.sh
- Dockerfile.mcp
  - 专门用于 MCP 服务器（HTTP 模式），暴露 3333 端口并以 HTTP 模式启动
- docker-compose.yml（官方示例）
  - 两个服务：trend-radar（爬虫+Web）、trend-radar-mcp（MCP 服务）
  - 端口映射：trend-radar 127.0.0.1:8080->8080（可由 WEBSERVER_PORT 覆盖），trend-radar-mcp 127.0.0.1:3333->3333
  - 卷挂载：config 读写、output 只读（MCP 服务）
  - 环境变量：时区、运行模式、推送配置、Web 服务器开关与端口、定时任务计划等
- docker-compose-build.yml（本地构建）
  - 从根目录 Dockerfile 构建镜像，其余配置与官方示例一致

### 容器入口与管理：entrypoint.sh 与 manage.py
- entrypoint.sh
  - 校验配置文件是否存在
  - 根据 RUN_MODE：
    - once：一次性执行 main.py
    - cron：生成 crontab 并由 supercronic 作为 PID 1 运行；可选立即执行一次；可选启动 Web 服务器
  - 保存环境变量至 /etc/environment
- manage.py
  - 提供 run/status/config/files/logs/restart/start_webserver/stop_webserver/webserver_status/help 等命令
  - Web 服务器默认端口 8080，绑定 0.0.0.0，限制在输出目录只读访问
  - 提供 cron 表达式的人类可读解析与容器状态检查

```mermaid
flowchart TD
EStart(["容器启动"]) --> CheckCfg["检查配置文件"]
CheckCfg --> Mode{"RUN_MODE"}
Mode --> |once| RunOnce["执行 main.py"]
Mode --> |cron| GenCron["生成 crontab 并校验"]
GenCron --> Immediate{"IMMEDIATE_RUN"}
Immediate --> |true| RunNow["立即执行一次"]
Immediate --> |false| Wait["等待定时触发"]
RunNow --> Wait
Wait --> WebCheck{"ENABLE_WEBSERVER"}
WebCheck --> |true| StartWS["启动 Web 服务器"]
WebCheck --> |false| End(["完成"])
StartWS --> End
```

## 故障排除指南
- 端口冲突
  - 症状：端口被占用导致服务无法启动
  - 解决：修改 docker-compose.yml 中的端口映射或释放宿主机端口
- 权限不足
  - 症状：容器无法写入输出目录或读取配置
  - 解决：确保挂载目录权限正确，容器内以非 root 用户运行时注意权限
- 配置缺失
  - 症状：entrypoint.sh 报告配置文件缺失
  - 解决：准备 config/config.yaml 与 frequency_words.txt，并确认挂载路径正确
- UV 安装失败（本地）
  - 症状：uv: command not found
  - 解决：手动添加到 PATH 或使用 pip 安装
- MCP 连接失败
  - 症状：客户端无法连接到 MCP 服务器
  - 解决：检查服务状态、端口占用、日志输出
