## 简介
TrendRadar 是一个“全网热点聚合与智能分析”系统，旨在帮助用户从海量信息中主动筛选出真正关心的热点与趋势。项目通过统一的数据采集、个性化排序与权重算法、多渠道实时推送以及基于 MCP 的 AI 对话分析能力，为不同角色用户提供“看得清、管得住、用得好”的信息决策支持。

- 核心目标
  - 聚合主流平台热点，去除平台算法干扰，按用户偏好重新排序
  - 提供三种推送模式（当日汇总、当前榜单、增量监控），满足不同使用场景
  - 支持多渠道通知（企业微信、飞书、钉钉、Telegram、邮件、ntfy、Bark、Slack）
  - 基于 MCP 的 AI 分析服务器，支持自然语言对话式查询与趋势洞察
  - 提供零门槛部署方案（GitHub 一键 Fork）与 Docker 容器化部署

- 目标用户
  - 数据分析师：通过 AI 工具进行跨平台对比、情感分析、相似新闻检索
  - 运营人员：监控竞品动态、品牌舆情、热点事件，辅助内容策略
  - 开发者与技术爱好者：快速搭建本地热点监控与分析流水线
  - 普通用户：减少无效刷屏，专注自身关注的领域

- 价值定位
  - 实时信息监控：统一抓取与清洗，稳定输出
  - AI 驱动分析：自然语言交互，降低分析门槛
  - 多渠道通知：消息直达手机与邮箱，避免错过关键信息
  - 可扩展生态：模块化设计，便于二次开发与集成

## 项目结构
项目采用“主程序 + MCP AI 分析服务 + 配置与部署”的分层组织方式，核心文件与目录如下：
- 根目录
  - README.md / README-EN.md：项目说明与使用指南
  - main.py：主程序，负责数据采集、关键词过滤、报告生成与推送
  - config/config.yaml：应用配置（报告模式、推送设置、平台权重、通知渠道等）
  - requirements.txt：Python 依赖
  - docker/docker-compose.yml：Docker 编排与环境变量
- mcp_server：AI 分析服务
  - server.py：MCP 服务器入口，注册 13+ 分析工具
  - tools/：分析工具集合（数据查询、智能检索、高级分析、系统管理）
  - services/：数据服务、缓存服务、解析服务
  - utils/：日期解析、参数校验、错误定义
- config/：配置文件与词表
- integrations/：第三方集成示例（如 Blogger 集成）
- docs/：部署与 API 参考文档
- 其他脚本：一键部署、启动脚本等

## 核心组件
- 数据采集与处理
  - 通过统一 API 获取多平台热搜，清洗标题、合并排名、统计频次
  - 支持请求重试、随机间隔、异常标题过滤，保障稳定性
- 关键词与筛选
  - 支持普通词、必须词、过滤词、数量限制、全局过滤等语法
  - 支持按热度优先或配置顺序优先排序
- 个性化热点算法
  - 通过“排名权重、出现频次权重、热度权重”三者组合，重新排序热点
  - 权重比例可调，满足不同场景偏好
- 推送策略与多渠道
  - 三种推送模式：当日汇总、当前榜单、增量监控
  - 支持推送时间窗口控制、内容顺序配置、多账号推送
  - 支持飞书、钉钉、企业微信、Telegram、邮件、ntfy、Bark、Slack
- AI 分析服务（MCP）
  - 提供 13+ 分析工具：基础查询、智能检索、趋势分析、情感分析、相似新闻、摘要生成、系统管理等
  - 支持自然语言日期解析，统一日期范围，确保分析一致性
  - 支持 STDIO/HTTP 两种传输模式，便于多种客户端接入

## 详细组件分析

### 数据采集与关键词过滤
- 数据采集
  - 通过统一 API 获取各平台热搜，按排名排序，合并相同标题，记录 URL 与移动端链接
  - 支持请求重试与随机间隔，避免平台限流
- 关键词过滤
  - 支持“必须词 +”、“过滤词 !”、“数量限制 @”、“全局过滤 [GLOBAL_FILTER]”
  - 支持按热度优先或配置顺序优先排序，灵活控制推送长度
- 个性化权重排序
  - 通过“排名权重、出现频次权重、热度权重”三者组合，重新排序热点，突出高价值内容


### 推送策略与多渠道
- 推送模式
  - 当日汇总：按时推送当日所有匹配新闻
  - 当前榜单：推送当前榜单匹配新闻
  - 增量监控：仅推送新增内容，零重复
- 多账号与分批
  - 所有推送渠道支持多账号配置，使用分号分隔
  - 对 Telegram/ntfy 等需要配对参数的渠道，自动校验数量一致性
  - 按渠道设置分批大小与间隔，避免消息超限
- 推送时间窗口
  - 可设置推送时间范围与“一天一次/窗口内多次”策略
  - 支持推送记录清理与留存天数控制

### AI 分析服务（MCP）
- 工具体系
  - 基础查询：获取最新新闻、按日期查询、趋势话题
  - 智能检索：关键词/模糊/实体搜索、历史相关新闻检索
  - 高级分析：话题趋势、数据洞察（平台对比/活跃度/关键词共现）、情感分析、相似新闻、摘要生成
  - 系统管理：获取当前配置、系统状态、手动触发爬取
- 日期解析
  - 统一将自然语言日期解析为标准日期范围，确保 AI 模型与系统日期一致
- 传输模式
  - 支持 STDIO（推荐）与 HTTP（生产）两种模式，便于不同客户端接入

## 故障排查指南
- GitHub Actions 限制
  - 由于 Fork 数量激增，官方对服务器压力较大，建议使用 Docker 部署
- 多账号配置
  - 多账号使用分号分隔；Telegram/ntfy 需保证配对参数数量一致；超过最大账号数会被截断
- 推送时间窗口
  - 若推送时间窗口设置不合理，可能导致推送记录异常；可通过清理记录或调整窗口范围解决
- MCP 服务连接
  - 确认 STDIO/HTTP 模式配置正确；使用 Inspector 测试连接；检查端口与路径
