## 通知渠道支持机制

TrendRadar 支持多种通知渠道，包括邮件、ntfy、Bark 和 Slack。这些渠道的配置均在 `config.yaml` 文件的 `notification` 部分进行。系统通过读取配置文件或环境变量来获取各渠道的认证信息和推送地址，并在满足条件时发送通知。

## 邮件通知配置

邮件通知通过 SMTP 协议发送，需要配置发件人邮箱、密码、收件人邮箱以及可选的 SMTP 服务器和端口。系统支持主流邮箱服务商的自动识别，如 Gmail、QQ 邮箱、Outlook 等。

### SMTP 认证配置

在 `config.yaml` 中，相关配置项如下：

```yaml
notification:
  webhooks:
    email_from: "" # 发件人邮箱地址
    email_password: "" # 发件人邮箱密码或授权码
    email_to: "" # 收件人邮箱地址，多个收件人用逗号分隔
    email_smtp_server: "" # SMTP服务器地址（可选，留空自动识别）
    email_smtp_port: "" # SMTP端口（可选，留空自动识别）
```

系统内置了常见邮箱服务商的 SMTP 配置，包括服务器地址、端口和加密方式。如果使用这些服务商，无需手动配置 `email_smtp_server` 和 `email_smtp_port`，系统会自动识别。

## ntfy通知配置

ntfy 是一个开源、免费且支持自托管的通知服务。TrendRadar 支持使用 ntfy 的公共服务或自托管实例。

### 自托管支持

在 `config.yaml` 中，相关配置项如下：

```yaml
notification:
  webhooks:
    ntfy_server_url: "https://ntfy.sh" # ntfy服务器地址，默认使用公共服务，可改为自托管地址
    ntfy_topic: "" # ntfy主题名称（多账号用 ; 分隔）
    ntfy_token: "" # ntfy访问令牌（可选，用于私有主题，多账号用 ; 分隔）
```

用户可以选择使用 ntfy 的免费公共服务，或在自己的服务器上部署 ntfy 实例以获得完全的隐私控制。自托管时，需将 `ntfy_server_url` 指向自己的服务器地址。

## Bark通知配置

Bark 是一款专为 iOS 平台设计的免费开源推送工具，特点是简洁高效。

### 设备密钥格式

在 `config.yaml` 中，`bark_url` 的格式为 `https://api.day.app/your_device_key` 或自建服务器地址。`your_device_key` 是在 Bark App 中生成的设备密钥。

```yaml
notification:
  webhooks:
    bark_url: "" # Bark推送URL（多账号用 ; 分隔，格式：https://api.day.app/your_device_key 或自建服务器地址）
```

用户可以通过下载 Bark App 并复制首页显示的推送 URL 来获取 `bark_url`。对于有服务器的用户，也可以自建 Bark 服务器。

## Slack通知配置

Slack 是一个团队协作工具，通过 Incoming Webhooks 可以将消息推送到 Slack 频道。

### Incoming Webhook集成方式

在 `config.yaml` 中，`slack_webhook_url` 的格式为 `https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXXXXXX`。

```yaml
notification:
  webhooks:
    slack_webhook_url: "" # Slack Incoming Webhook URL（多账号用 ; 分隔，格式：https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXXXXXX）
```

集成步骤包括：创建 Slack App、启用 Incoming Webhooks、生成 Webhook URL 并将其配置到 TrendRadar 中。

## 消息分批与多账号策略

TrendRadar 支持对消息进行分批发送，并允许为每个渠道配置多个账号。

### 消息分批大小

在 `config.yaml` 中，可以设置各渠道的消息分批大小：

```yaml
notification:
  bark_batch_size: 4000 # Bark消息分批大小（字节）
  slack_batch_size: 4000 # Slack消息分批大小（字节）
```

分批大小以字节为单位，确保单条消息不会超过服务的限制。

### 多账号分号分隔策略

所有支持多账号的渠道（如飞书、钉钉、Telegram、ntfy、Bark、Slack）均使用英文分号 `;` 分隔多个账号的值。例如：

```yaml
notification:
  webhooks:
    feishu_url: "url1;url2;url3"
    ntfy_topic: "topic1;topic2"
    bark_url: "url1;url2"
    slack_webhook_url: "url1;url2"
```

对于需要配对的配置（如 Telegram 的 token 和 chat_id，ntfy 的 topic 和 token），数量必须一致。

## 服务注册与集成实践

### 邮件服务

1. **选择邮箱服务商**：支持 Gmail、QQ 邮箱、Outlook 等主流服务商。
2. **获取授权码**：在邮箱设置中开启 POP3/SMTP 服务并生成授权码。
3. **配置 `config.yaml`**：填写 `email_from`、`email_password` 和 `email_to`。

### ntfy服务

1. **选择使用方式**：
   - **免费使用**：使用 ntfy.sh 作为服务器，订阅一个难猜的主题名称。
   - **自托管**：使用 Docker 一键部署 ntfy 服务器。
2. **配置 `config.yaml`**：填写 `ntfy_server_url`、`ntfy_topic` 和可选的 `ntfy_token`。

### Bark服务

1. **下载 Bark App**：从 App Store 下载并安装。
2. **获取推送 URL**：打开 App，复制首页显示的推送 URL。
3. **配置 `config.yaml`**：将 URL 填入 `bark_url`。

### Slack服务

1. **创建 Slack App**：在 Slack API 页面创建新应用。
2. **启用 Incoming Webhooks**：在应用设置中激活 Incoming Webhooks 功能。
3. **生成 Webhook URL**：添加新的 Webhook 并选择目标频道。
4. **配置 `config.yaml`**：将生成的 URL 填入 `slack_webhook_url`。
